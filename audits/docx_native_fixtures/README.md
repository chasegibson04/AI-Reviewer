# DOCX-Native Fixture Test Bed

This fixture set is derived only from the approved validation projects:

- `20260325163524_test-existingphactorpaper`
- `20260327051312_miniaturization_d2b`

The horseshoe crab project is excluded completely.

## Fixture Types

For each approved project the generator creates:
- `clean_native`
- `commented_light`
- `commented_heavy`
- `prior_ai_commented`
- `prior_ai_suggested`

## Source Policy

Fixtures are generated by:
- [build_docx_native_fixtures.py](C:\Users\Cernak.DESKTOP-NJGGBU3\Desktop\Chase\D2B TOOLS\AI-Reviewer\scripts\build_docx_native_fixtures.py)

Source choices:
- `clean_native` prefers the project's managed native DOCX
- if no managed native DOCX exists, the generator falls back to the safest approved-project-derived DOCX base
- `commented_light` and `commented_heavy` add synthetic human/editor comments
- `prior_ai_commented` is copied from prior AI-Reviewer commented output
- `prior_ai_suggested` is copied from prior AI-Reviewer suggested-revision output

Manifest written to:
- `audits/docx_native_fixtures/generated/fixture_manifest.json`

## What These Fixtures Validate

- clean native DOCX parsing
- existing comment preservation
- pre-annotated DOCX re-review
- prior suggested-block stripping from analysis text
- no-op detection via `meaningful_new_review_state` and `silent_noop_suspected`
