# DOCX-Native Fixture Test Bed

This test bed is derived only from the approved projects:

- `20260325163524_test-existingphactorpaper`
- `20260327051312_miniaturization_d2b`

It excludes the horseshoe crab project completely.

## Fixture Types

For each approved project the generator creates:

- `clean_native`
- `commented_light`
- `commented_heavy`
- `prior_ai_commented`
- `prior_ai_suggested`

## How Fixtures Are Built

Fixtures are generated by:

- [build_docx_native_fixtures.py](C:\Users\Cernak.DESKTOP-NJGGBU3\Desktop\Chase\D2B TOOLS\AI-Reviewer\scripts\build_docx_native_fixtures.py)

Source material:

- `clean_native` starts from the project's surrogate base DOCX generated from approved project content.
- `commented_light` and `commented_heavy` are created by adding synthetic reviewer/editor comments across multiple sections.
- `prior_ai_commented` is copied from a prior AI-Reviewer commented DOCX run artifact.
- `prior_ai_suggested` is copied from a prior AI-Reviewer suggested-changes DOCX run artifact.

The generated fixture manifest is written to:

- `audits/docx_native_fixtures/generated/fixture_manifest.json`

## Fixture Goals

These fixtures test:

- clean native DOCX handling
- native DOCX with existing comments
- native DOCX with many prior comments
- re-review of prior AI-Reviewer commented output
- re-review of prior AI-Reviewer suggested-change output
